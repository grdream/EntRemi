You are an expert Laravel 13 full-stack developer. Your first task is to create a detailed `plan.md` file before writing any code. This plan will guide the entire development of a WatchList Reminder Web Application built with Laravel 13, MySQL, Blade + Alpine.js + Livewire, and TailwindCSS.

---

## PROJECT OVERVIEW

Build a full-featured WatchList Reminder Application where users can:
- Register and login with their own accounts
- Add dramas, movies, anime, TV series, and any watchable content to their personal watchlist
- Auto-fetch content details (poster, description, episodes, air dates) from free APIs
- Set smart episode schedule patterns (daily, weekly, bi-weekly, custom, etc.)
- Receive email (SMTP) and SMS reminders before new episodes air
- Manage all notification preferences from their dashboard
- View notification history and upcoming episode schedule

---

## TECH STACK

- **Framework:** Laravel 13
- **PHP:** 8.3+
- **Database:** MySQL 8.0+
- **Frontend:** Blade templates + Alpine.js + Livewire 3 + TailwindCSS 3
- **Auth:** Laravel Breeze (Blade stack)
- **Mail:** SMTP via Laravel Mail (config via .env — NOT php mail())
- **SMS:** Custom HTTP gateway integration (ViserLab SMSLab PHP script via cURL POST)
- **Queue/Jobs:** Laravel Queue (database driver, upgradeable to Redis)
- **Scheduler:** Laravel Task Scheduler (cron every 15 minutes)
- **APIs Used (all free):**
  - TMDB API (movies, TV series, dramas) — free API key required
  - Jikan API v4 (anime) — completely free, no key needed
  - YouTube oEmbed API (fetch title + thumbnail from YouTube links) — free, no key
  - Open Library / manual entry fallback

### PHASE 5 — Content Search & Auto-Fetch System
Build the content discovery system:

**Search Flow:**
1. User goes to "Add to Watchlist"
2. Sees tabs: Search Online | Paste YouTube Link | Manual Entry
3. **Search Online tab:**
   - Single search box with type filter (All / Movie / TV Series / Drama / Anime)
   - Live search using Livewire (debounce 500ms)
   - For non-anime: hits TMDB API `/search/multi` endpoint
   - For anime: hits Jikan API `/anime` endpoint
   - Results show as cards: poster, title, year, type, rating, brief description
   - Click "Add to Watchlist" on any result

4. **Paste YouTube Link tab:**
   - User pastes a YouTube URL (episode or trailer)
   - Fetch via YouTube oEmbed: `https://www.youtube.com/oembed?url={url}&format=json`
   - Returns: title, thumbnail_url, author_name
   - Auto-fill the manual entry form with fetched data

5. **Manual Entry tab:**
   - Full form: title, type, description, poster URL or upload, total episodes, year, country, language, genres

**TMDB Integration:**
- Store TMDB_API_KEY in .env
- Create App\Services\TmdbService class
- Methods: search($query, $type), getMovieDetails($tmdbId), getTvDetails($tmdbId), getEpisodes($tmdbId, $season)
- Cache API responses for 24 hours using Laravel Cache

**Jikan Integration:**
- Create App\Services\JikanService class
- Methods: search($query), getAnimeDetails($jikanId), getEpisodes($jikanId)
- No API key needed
- Cache responses for 24 hours

**YouTube oEmbed Integration:**
- Create App\Services\YoutubeService class
- Method: fetchDetails($url) — returns title, thumbnail, channel

Expected: User can search and add any content. Poster and details auto-fill.

### PHASE 6 — Episode Scheduler & Smart Schedule Engine
This is the most complex phase. Build the episode scheduling system.

**When adding a show, after content details are fetched, show a Schedule Setup form:**

Schedule patterns (user selects one):
1. **Daily** — one new episode every day at [time]
2. **Weekly (1 episode)** — every [day of week] at [time]
3. **Weekly (2 episodes same day)** — every [day] at [time1] and [time2]
4. **Bi-weekly** — every [day1] and [day2] at [time]
5. **Twice per week (different times)** — [day1] at [time1] and [day2] at [time2]
6. **Monthly** — on day [X] of each month at [time]
7. **One-time / Movie** — single date and time picker (for movies or special episodes)
8. **Irregular / Manual** — user adds episode air dates one by one

**Schedule Engine (App\Services\ScheduleEngine):**
- Method: generateEpisodes(Show $show, Schedule $schedule, int $fromEpisode = 1)
- Auto-calculates all future episode air_datetime values from pattern
- Stores each as an Episode record in the database
- If total_episodes is known: generate all episodes
- If unknown (ongoing): generate next 30 episodes, re-run weekly via scheduler
- Handles timezone conversion (user timezone → UTC for storage)
- Method: regenerateFromEpisode(Show $show, int $episodeNo) — for when user updates schedule

**Per-show reminder overrides:**
- On each show's detail page, user can override global reminder settings
- Set custom reminder timing just for that show
- Enable/disable notifications just for that show

**Episode management page per show:**
- List all episodes with air dates
- Mark episode as aired manually
- Edit individual episode's air datetime
- Add a custom note per episode
- Attach YouTube link to any episode

Expected: After adding a show with a schedule, all episode dates are auto-generated and visible.

### PHASE 7 — Reminder Engine & Notification Dispatch
Build the core notification system.

**Laravel Scheduler (app/Console/Kernel.php or routes/console.php in Laravel 13):**
- Every 15 minutes: run CheckUpcomingEpisodes command
- Daily at midnight: run GenerateUpcomingEpisodes command (generates next 30 episodes for ongoing shows)
- Daily at 6am: run SendDailySummary command (optional digest email)

**CheckUpcomingEpisodes Command (App\Console\Commands\CheckUpcomingEpisodes):**
- Query all episodes where:
  - is_aired = false
  - notified = false
  - air_datetime is within the next [remind_before_minutes] window
  - Show has active reminders
  - User has notifications enabled
- For each matching episode:
  - Dispatch SendEpisodeReminderJob to the queue
  - Mark episode as notified = true

**SendEpisodeReminderJob (App\Jobs\SendEpisodeReminderJob):**
- Receives: Episode, User, channels (email, sms)
- For email channel:
  - Check if user has custom SMTP settings → use those
  - Fallback to .env SMTP if no custom settings
  - Send using App\Mail\EpisodeReminderMail Mailable
  - Beautiful HTML email: show poster, episode number, title, air time, countdown
- For SMS channel:
  - Check if user has SMS settings and sms_notifications enabled
  - Build message string: "Reminder: [Show Name] Episode [X] airs in [Y] minutes! Don't miss it."
  - Fire cURL POST to user's gateway URL with their params
  - Handle response and log success/failure
- Log result to notification_logs table

**App\Mail\EpisodeReminderMail:**
- Blade email template (resources/views/emails/episode-reminder.blade.php)
- Responsive HTML email design
- Show poster image, show title, episode number, episode title (if known)
- Air time in user's timezone
- Countdown (e.g. "Airs in 30 minutes!")
- Footer with unsubscribe-style link to notification settings

**App\Services\SmsService:**
- Method: send(User $user, string $message): bool
- Reads user's UserSmsSetting
- Builds POST params (gateway_url, api_key, sender_id, extra_params + phone + message)
- Fires cURL POST
- Returns true/false based on HTTP response

Expected: Reminders fire automatically. Emails and SMS sent at correct times.

## IMPORTANT RULES FOR THE AI AGENT

1. **Always read plan.md before starting any phase** — follow it exactly
2. **Create plan.md FIRST before writing any code**
3. **Use Laravel 13 syntax** — use PHP 8.3 attributes where applicable, use routes/console.php for scheduler (not Kernel.php which is removed in Laravel 11+)
4. **SMTP mail is the ONLY mail driver** — never use PHP mail() function. Use Laravel's Mail facade with runtime SMTP config from user_smtp_settings table. Fallback to .env SMTP credentials.
5. **SMS is disabled by default** — user must explicitly enable it in settings
6. **Email is enabled by default** — user can disable it in settings
7. **All sensitive data encrypted** — SMTP passwords and SMS API keys use Laravel Crypt::encrypt() before storing and Crypt::decrypt() when reading
8. **Queue everything** — never send emails or SMS synchronously in a request. Always dispatch a Job.
9. **Timezone aware** — always store datetimes in UTC, display in user's local timezone
10. **Mobile responsive** — all UI must be fully responsive with TailwindCSS
11. **Livewire for dynamic UI** — search, toggles, real-time updates use Livewire 3 components
12. **Alpine.js for micro-interactions** — countdown timers, dropdowns, modals use Alpine.js
13. **Cache API responses** — TMDB and Jikan responses cached for 24 hours using Laravel Cache
14. **Error handling** — all external API calls and SMS/email sends wrapped in try/catch, errors logged
15. **Soft deletes on shows** — so user can restore accidentally deleted shows
16. **No hardcoded API keys** — everything in .env

### PHASE 7 — Reminder Engine & Notification Dispatch
Build the core notification system.

**Laravel Scheduler (app/Console/Kernel.php or routes/console.php in Laravel 13):**
- Every 15 minutes: run CheckUpcomingEpisodes command
- Daily at midnight: run GenerateUpcomingEpisodes command (generates next 30 episodes for ongoing shows)
- Daily at 6am: run SendDailySummary command (optional digest email)

**CheckUpcomingEpisodes Command (App\Console\Commands\CheckUpcomingEpisodes):**
- Query all episodes where:
  - is_aired = false
  - notified = false
  - air_datetime is within the next [remind_before_minutes] window
  - Show has active reminders
  - User has notifications enabled
- For each matching episode:
  - Dispatch SendEpisodeReminderJob to the queue
  - Mark episode as notified = true

**SendEpisodeReminderJob (App\Jobs\SendEpisodeReminderJob):**
- Receives: Episode, User, channels (email, sms)
- For email channel:
  - Check if user has custom SMTP settings → use those
  - Fallback to .env SMTP if no custom settings
  - Send using App\Mail\EpisodeReminderMail Mailable
  - Beautiful HTML email: show poster, episode number, title, air time, countdown
- For SMS channel:
  - Check if user has SMS settings and sms_notifications enabled
  - Build message string: "Reminder: [Show Name] Episode [X] airs in [Y] minutes! Don't miss it."
  - Fire cURL POST to user's gateway URL with their params
  - Handle response and log success/failure
- Log result to notification_logs table

**App\Mail\EpisodeReminderMail:**
- Blade email template (resources/views/emails/episode-reminder.blade.php)
- Responsive HTML email design
- Show poster image, show title, episode number, episode title (if known)
- Air time in user's timezone
- Countdown (e.g. "Airs in 30 minutes!")
- Footer with unsubscribe-style link to notification settings

**App\Services\SmsService:**
- Method: send(User $user, string $message): bool
- Reads user's UserSmsSetting
- Builds POST params (gateway_url, api_key, sender_id, extra_params + phone + message)
- Fires cURL POST
- Returns true/false based on HTTP response

## IMPORTANT RULES FOR THE AI AGENT

1. **Always read plan.md before starting any phase** — follow it exactly
2. **Create plan.md FIRST before writing any code**
3. **Use Laravel 13 syntax** — use PHP 8.3 attributes where applicable, use routes/console.php for scheduler (not Kernel.php which is removed in Laravel 11+)
4. **SMTP mail is the ONLY mail driver** — never use PHP mail() function. Use Laravel's Mail facade with runtime SMTP config from user_smtp_settings table. Fallback to .env SMTP credentials.
5. **SMS is disabled by default** — user must explicitly enable it in settings
6. **Email is enabled by default** — user can disable it in settings
7. **All sensitive data encrypted** — SMTP passwords and SMS API keys use Laravel Crypt::encrypt() before storing and Crypt::decrypt() when reading
8. **Queue everything** — never send emails or SMS synchronously in a request. Always dispatch a Job.
9. **Timezone aware** — always store datetimes in UTC, display in user's local timezone
10. **Mobile responsive** — all UI must be fully responsive with TailwindCSS
11. **Livewire for dynamic UI** — search, toggles, real-time updates use Livewire 3 components
12. **Alpine.js for micro-interactions** — countdown timers, dropdowns, modals use Alpine.js
13. **Cache API responses** — TMDB and Jikan responses cached for 24 hours using Laravel Cache
14. **Error handling** — all external API calls and SMS/email sends wrapped in try/catch, errors logged
15. **Soft deletes on shows** — so user can restore accidentally deleted shows
16. **No hardcoded API keys** — everything in .env
